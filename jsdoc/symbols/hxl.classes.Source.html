{hxl.classes.Source} cache()
Cache the transformed HXL for future use. Create a copy of the HXL dataset, as transformed up to this point in the filter chain, to save future work. Some filters, like hxl.classes.CountFilter, already build their own cache, and don't need this filter. 
var filtered = data.withRows('sector=WASH').withoutColumns('contact').cache();
Returns:
{hxl.classes.Source} a new data source, with all transformations cached.

{int} each(callback)
Fire a callback on each row of data. The callback has the form

(Often, the callback will need to bother with only the first parameter, so function (row) {} will be more typical).

{object} iterator()
Get an iterator to read through the rows of a HXL dataset. The iterator will have a next() method to read each row in sequence, and will return null once the rows are exhausted. 
var row;
var iterator = data.iterator();
while (row = iterator.next()) {
  // do something with the row
}
The #each or #forEach method provides a more-elegant way of iterating through rows, but this method makes it easier to break out of the loop early.
Returns:
{object} An iterator object with a .next() method.

{hxl.classes.Source} rename(pattern, newTag, newHeader, index)
Change the tag (and optionally, header) for one or more columns. Use this to change around tags and attributes. For example, you could change the first instance of #org to #org+funder to make the data easier to filter later: 
var filtered = data.rename('#org', '#org+funder', 'Donor', 0);
Parameters:
{string} pattern
The tag pattern to match for replacement. See hxl.classes.TagPattern#parse for details.
newTag
the new HXL tag spec (e.g. "#adm1+code"). See {@link hxl.classes.Column#parse} for details.
{string} newHeader
(optional) The new header. If undefined, don't change.
{int} index
The zero-based index to replace among matching tags. If undefined, replace *all* matches.
Returns:
{hxl.classes.Source} A new data source, with matching column(s) replaced.
